# 系统更新中心与集群更新

“系统更新中心”是当前版本比较新的能力，它把“检查版本”“发起更新”“查看任务”“管理节点”放到了设置页高级区里。

## 适用对象

- 管理员
- 有多节点或集群更新需求的部署环境

## 页面结构

### 总览配置

- 当前版本
- 最新版本
- 最近检查
- 当前状态
- 版本源配置
- 默认更新范围与策略
- 创建更新任务

### 任务执行

- 看活跃任务
- 看批次进度
- 看历史任务
- 支持失败重试或继续跟踪

### 节点状态

- 看更新代理心跳
- 管理节点排空
- 恢复接流

## 当前支持的范围与策略

### 范围

- `app`
- `worker`
- `cluster`

### 策略

- `in_place`
- `rolling`
- `drain_and_cutover`

## 排空切换的意义

排空切换不是简单重启，而是先把任务从节点上迁走，再做更新，尽量降低影响范围。

当前版本已经支持：

- 节点排空
- 调度器避开排空节点
- 排空后恢复接流

但仍要注意：

- 账号运行态热迁移不是所有场景都已完全无感
- 如果账号凭据只剩一次性登录态，切换后仍可能需要重新登录

## 宿主机更新代理

系统更新中心依赖宿主机上的更新代理脚本协同工作。

### 单次执行

```bash
/opt/qq-farm-current/update-agent.sh --once
```

### 常驻服务

```bash
/opt/qq-farm-current/install-update-agent-service.sh
```

## 建议

- 单机环境可以把它当成“可视化更新入口”。
- 多节点环境要先明确每个代理托管了哪些节点，再创建任务。
- 真正执行 `drain_and_cutover` 前，先看预检结果，不要跳过阻塞信息。
