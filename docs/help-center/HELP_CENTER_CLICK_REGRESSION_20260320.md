# 帮助中心真实点击回归记录

## 背景

本轮回归针对帮助中心和页面上下文帮助入口补了一套真实点击验证，目标不是只验证静态渲染，而是确认以下关键链路在浏览器里可实际完成：

- 文档切换
- 正文搜索
- 命令复制
- 治理任务复制
- 设置页上下文帮助跳转
- 帮助中心返回设置页具体分组 / 锚点

## 覆盖场景

### 场景 1：文章切换与正文搜索

- 进入帮助中心并定位到 `设置页总览`
- 在帮助中心搜索框输入 `排空`
- 点击搜索结果 `系统更新中心与集群更新`
- 验证文章标题与 URL `article` 参数同步切换

### 场景 2：命令复制与治理任务复制

- 进入 `更新、修复与回滚`
- 点击命令块右上角 `复制命令`
- 验证剪贴板得到非空脚本文本
- 点击 `复制治理任务`
- 验证剪贴板包含治理模板标题与治理单号

### 场景 3：设置页锚点帮助跳转与返回

- 进入设置页高级设置的系统更新区域，并通过 query/hash 定位到 `任务执行`
- 点击 `当前分类帮助`
- 验证帮助中心 URL 保留 `article`、`audience`、`section`
- 验证帮助中心目录高亮到 `任务执行`
- 点击帮助中心 `关联页面` 中的 `更新中心 · 任务执行`
- 验证返回设置页时恢复 `category=advanced`、`advancedSection=update`、`updateTab=jobs` 与对应 hash

## 本轮发现并修复的问题

### 帮助中心初始化会丢失外部传入的 `section`

表现：

- 从设置页上下文帮助跳进帮助中心时，URL 初始带了 `section`
- 帮助中心 `onMounted` 再次同步文章状态时，把 `section` 重置为空
- 导致帮助文章虽然打开了，但不会滚动到预期分组

处理：

- 初始化时保留 `route.query.section`
- 同步当前文章时不再覆盖已有锚点参数

### 回归环境里首屏版本提示会遮挡复制按钮

表现：

- 自动回归点击命令复制按钮时，被首屏更新提示层拦截

处理：

- 回归里预置已读版本状态
- 保留显式关闭提示层的兜底逻辑

### 设置页高级区依赖的种子接口未被 mock

表现：

- 设置页高级区在回归环境里加载时，把接口返回对象当成数组使用，触发 `map is not a function`

处理：

- 为 `/api/seeds` 与 `/api/bag/plantable-seeds` 提供数组型 mock
